# AI Tooling

This project was developed with AI-assisted engineering support.

## Tools used

- GPT-based coding assistant for implementation and refactoring
- AI assistance for UI adjustments, route design, and documentation drafting

## What worked well

- speeding up iteration on admin UI and widget behavior
- generating consistent TypeScript refactors across related files
- drafting project documentation and evaluation scaffolding
- identifying requirement gaps against the project brief

## What needed human review

- verifying that RAG guardrails matched the project requirements
- confirming route behavior and UI flow in the actual app
- checking that citations and status transitions were correct
- deciding which project requirements were code changes versus manual deliverables

## Limitations observed

- AI suggestions can optimize locally while missing higher-level rubric requirements
- broad edits need manual scoping to avoid changing established workflows
- generated evaluation logic still requires human judgment for final interpretation
